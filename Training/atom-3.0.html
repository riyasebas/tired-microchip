Context

Why now

The bar for what an AI assistant should feel like has already been set — by Claude, ChatGPT, and every consumer AI tool in the market. People expect real-time responses, transparency into what the AI is doing, and conversations that understand context.

When they come to work and interact with an AI assistant that's slower and less conversational, the gap feels obvious. We've heard this sentiment echoed by customers and prospects. Atom needs to be smarter, faster, and more reliable; the conversational experience needs to feel modern.

Atom 3.0 closes that gap — and then goes further.

This isn't just about catching up to expectations. It's about what's now possible. The underlying AI models have taken a generational leap in reasoning and reliability. Atom can now understand vague requests without guessing wrong, execute multi-step actions reliably, and adapt its behavior to each customer's environment.

This also lays the foundation for where we're headed — an AI workforce of specialized agents that don't just answer questions but actively resolve issues, triage work, and operate across systems.

These upgrades make everything work better — they don't redesign how people use the product.

Atom understands what users actually mean

We've made this claim before. What's changed now is the foundation underneath it.

Earlier, clarifying questions were prompt-engineered rules — Atom was explicitly told when to ask them and when not to. This is why sometimes it felt excessive or robotic. Now, Atom runs on a reasoning-based model that actually works through ambiguity before responding. It doesn't ask clarifying questions because it was told to — it asks them because it understands why clarification is needed. It handles multiple questions in one message. It distinguishes between similar catalog items. And it follows up more accurately in longer conversations.

Disambiguation in action: When someone types "I'm locked out," Atom identifies that there are multiple possible meanings, evaluates what it knows about the user, and asks a specific question: "Are you locked out of the physical office, or a specific app?"

Multi-part queries handled cleanly: When a user asks about both the holiday calendar and the remote work policy in one message, Atom identifies both questions, responds to each separately, and clearly states what it found versus what it couldn't find.

Verified answers: When a verified answer exists, Atom surfaces it and will try to explicitly call it out — e.g., "Here's an FAQ I found" — so users know the response is coming from a trusted, admin-curated source rather than a general search.

Web search: Atom searches trusted external sources (like vendor documentation) alongside internal knowledge. This is configurable per customer.

Configurable per customer: We can now tune Atom's behavior at the tenant level — tone, clarification depth, fallback behavior, how aggressively it disambiguates. A customer who wants Atom to ask more clarifying questions before resolving can have that. One who wants minimal friction can have the opposite. Today this is managed by our team; self-service configuration for admins is coming soon.

Atom doesn't make things up anymore

When Atom doesn't have the answer, it now clearly says so instead of generating a plausible-sounding but wrong response.

So when asked "What is our remote work policy? Can I work from another country for three weeks?" — and no such policy exists in the knowledge base — a likely response will be: "I couldn't find a company-specific remote work policy. I can't confirm whether you're allowed to work from another country."

We can expect up to a 50% reduction in hallucinated responses.

Atom is more reliable when taking actions

Previously, Atom used an in-house orchestration layer to execute actions — lookups, request creation, automations. There was a translation step between the AI model understanding what to do and actually doing it. That's where things would break: Atom understood the request but fumbled the execution.

Atom now connects to tools directly, which means the model itself decides what to call, with what inputs, and in what order. The result: actions execute more consistently, with fewer errors, fewer "I couldn't understand that" moments, and fewer failed executions.

Speed & data privacy

Response times are 20–25% faster and significantly more consistent. Users will notice the difference, especially on questions that previously felt sluggish.

Data privacy is locked down by default. All AI processing uses enterprise-grade infrastructure with zero data retention. No employee data is stored, logged, or used for model training. This is our default configuration for every tenant.

Major Upgrade 2

Upgrading the Atom portal experience

Atom's portal experience is getting significant upgrades — a personalized landing page that helps employees get started faster. This is especially important for customers on Microsoft Teams, since the full portal experience will be embedded directly inside Teams.

Personalized prompts replace the blank chat window

When employees open Atom on the portal or the browser extension, they now see personalized prompts instead of a blank chat window.

  • Scoped to permissions: Prompts are pre-generated from the tenant's published knowledge articles and service catalogs, scoped to what the user is allowed to see. Unpublished catalogs or restricted content won't surface as prompts.
  • Contextual to the employee: First-time employees see prompts around knowledge, incidents, and service requests — things like "How do I request a new laptop?" or "What's the guest Wi-Fi password?" Returning employees see prompts about their active requests first (e.g., "Check status of my Salesforce access request"), so they can pick up where they left off with one click.
  • Role-level personalization Coming soon: Prompts tailored based on the user's role or function, informed by what teammates in the same role are asking. This is built and architected but not yet enabled.
  • Catalog discovery Coming soon: Surfacing relevant catalog items (services employees can request) as prompts, not just knowledge articles.
Major Upgrade 2 (continued)

Embedding the Atom portal into Microsoft Teams

We're fundamentally transforming how Atom works inside Microsoft Teams. Instead of the current native app experience — where users interact with Atom in a standard Teams chat thread — we're embedding the full Atomicwork portal directly into Teams, so users get our complete portal experience without leaving Teams.

Why this matters

The current Teams experience depends on Microsoft's messaging infrastructure — one long, never-ending thread. There's no way to separate conversations, no threading, and no easy way to go back and reference past interactions. It also can't properly support the rich interactions Atom is capable of or show our full range of UI elements.

By embedding the portal, we solve both problems. Users get a structured, full-featured experience — and as long as Atomicwork's platform is up, it works.

The three tabs

💬

Chat

Full portal chat interface. Conversation history in a collapsible sidebar, ability to start new conversations, inline notifications. Identical to the portal and browser extension.

🔔

Notifications

All notifications in one place: request updates, change comments, approvals, workflow status. Also pushes to Teams Activity feed for desktop and mobile alerts.

✅

Approvals

A dedicated tab for all active approvals. Approvers manage everything in one place instead of digging through chat history.

What users should know

  • Authentication is seamless: The app inherits the user's existing Microsoft Teams login — no separate sign-in required.
  • Custom branding carries over: Your branded experience — Atom's name, the avatar — carries over to the embedded portal.
  • Requests are not affected: All existing requests, statuses, and associated data carry over completely. This change only affects the conversational interface.
  • Request history carries over: All requests from the past 6 months will be accessible through the Notifications tab.
  • Dark mode and mobile: The embedded portal supports dark mode (inherits device-level settings) and is mobile responsive — the same app works on Teams mobile.
  • Coming soon — voice and vision: The initial launch does not include real-time voice and camera capabilities in Teams due to Microsoft SDK requirements. Users can still send images and screenshots alongside their messages.

Major Upgrade 3

Streaming: Real-time responses and reasoning trails

Atom is getting native streaming across all channels. Instead of waiting for the entire response to be generated before displaying it, answers will flow in real time — word by word — just like ChatGPT, Claude, or any modern AI assistant.

Alongside streaming, Atom will show live reasoning trails: what it's doing, what it's searching, what it found — all visible as the response is being generated.

Why it matters

Today, when a user asks Atom a question, they see a "Thinking..." indicator and then the full response appears all at once. For complex queries that take a few seconds, this creates a perception of slowness even when the actual processing time is reasonable.

Streaming changes this. Users see Atom responding immediately — text flowing in progressively. Combined with the intelligence layer upgrade that already makes responses 20–25% faster, the experience will feel dramatically more responsive.

What it looks like

Streaming responses: When a user sends a message, they immediately see Atom begin responding. Text flows in progressively, mirroring what users now expect from any AI-powered tool.

"Show your work" reasoning trails: Users see what Atom is doing in real time: "Searching knowledge base..." → "Found 3 relevant articles..." → "Checking your service catalog..." → the actual response. This transparency builds trust.

For admins, this goes deeper. They can click into any step in the reasoning trail and see exactly what Atom did — for example, if Atom triggered a password reset, the admin can see the exact action, its inputs, and the result. This is a significant improvement for troubleshooting and audit.

Long-running task indicators: For actions that take longer — like waiting for an approval — the conversation won't freeze. Users see an in-progress status indicator, can continue chatting with Atom, and the status resolves gracefully when the action completes.

Redesigned conversation view: The Request Details page will include a Conversations tab showing the complete conversation flow including the reasoning trail (so admins can see what steps Atom took), and buttons/CTAs that were presented to the user.

Important: Streaming is a presentation-layer change. It changes how responses appear on screen but doesn't change how Atom thinks or generates answers. The intelligence layer handles all reasoning. Accuracy, action reliability, and all the improvements from the intelligence upgrade remain unchanged.

FAQs

Will there be any disruption or downtime during any of the upgrades?
No. The upgrades are seamless — no downtime, no user action required, no interruption to service. We manage the cutover on our side and it's designed to be invisible to end users.
How do we know quality won't regress?
Two things. First, we test every engine change internally against a comprehensive set of real-world questions based on actual customer usage. Our benchmarks show ~50% accuracy improvement and 20–25% latency improvement.

Second, customers can measure the improvement themselves using the Test Suite. Create a baseline dataset now, re-run it after the upgrade, and see the results firsthand.
Atom sometimes makes up answers — is that fixed?
Atom now gives significantly higher priority to verified answers and ingested topics. When it doesn't have the answer, it explicitly says so. Benchmarks show roughly a 50% reduction in fabricated answers.
What if a user asks two unrelated questions in the same message?
Atom is now significantly better at handling multi-part queries — it identifies both questions and responds to each. The one exception is access requests using IGA, where the access flow can take priority. We recommend users submit access requests separately for the best experience.
Does streaming affect accuracy?
Not at all. Streaming only changes how the response appears on screen — progressively instead of all at once. The intelligence layer handles all reasoning. It's purely a presentation-layer change.
Is any customer data being sent to OpenAI?
Atom processes queries through OpenAI's enterprise infrastructure with zero data retention — nothing is stored, logged, or used for training. This is the highest security tier available and it's our default for all tenants.
Can customers control how Atom behaves for their tenant?
Yes. We can now configure Atom's behavior per customer — tone, clarification depth, fallback behavior, disambiguation. Today this is managed by the Atomicwork team. We're building toward a self-service model where admins can edit Atom's behavior through plain-English instruction files.
Are there new security implications with the Teams embed?
No. Same SSO, same token-based access, encrypted communication throughout. No new security surface is introduced.
What about conversation history when switching to the new Teams experience?
All conversations from the last 6 months will be available. All requests and their full data are completely unaffected. The only thing that changes is the conversational interface.
What if customers don't want to upgrade the Teams experience yet?
That's fine — the update only applies when the admin explicitly clicks "Update Now." There's no auto-push. The current experience will continue to work.
